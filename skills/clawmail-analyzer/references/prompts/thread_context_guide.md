# 线程上下文分析指南

当邮件属于一个对话线程时，你会收到此前邮件的一句话摘要作为上下文。

## 分析要求

1. **聚焦增量内容**: 摘要不要重复线程中已有信息。只概括当前邮件**新增**的内容。
2. **理解上下文**: 利用线程历史理解当前邮件的背景。如果当前邮件说"同意"或"OK"，结合上下文理解同意的是什么。
3. **追踪决策链**: 如果线程包含多轮讨论，在 key_points 中总结最新的决策或结论。
4. **行动项去重**: 不要提取线程中已出现的行动项，只提取当前邮件新增的。
5. **情感变化**: 如果当前邮件的情感与此前线程不同，在摘要中体现这一变化。

## one_line 要求

对于线程回复邮件，one_line 应体现"回复了什么"而非重复原始主题。

好的例子：
- "张总同意了推迟发布的方案"
- "李四补充了预算表的缺失数据"
- "王五拒绝了加班安排并提出替代方案"

差的例子：
- "关于Q4发布的讨论"（太笼统，没有增量信息）
- "回复：Q4项目进度汇报"（只是重复了主题行）
