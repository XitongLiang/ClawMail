# 摘要规范详解

## 摘要生成原则

### 1. 信息密度
- 每句话都要携带有效信息
- 避免冗余和重复
- 省略礼貌用语和客套话

### 2. 结构清晰
- 使用固定模板
- 分层级展示信息
- 重要信息前置

### 3. 客观准确
- 不添加主观判断
- 不夸大或缩小事实
- 保留关键数字和时间

## 各部分详解

### 【发件人】
格式：`姓名/组织 + 职位/角色`

示例：
- ✅ 张三 - 项目经理
- ✅ ABC公司 - 销售部
- ✅ 李四 <lisi@example.com>

### 【核心主题】
要求：一句话概括，不超过 50 字

示例：
- ✅ 关于Q4季度预算审批的会议邀请
- ✅ 产品上线延迟通知及后续安排
- ❌ 这封邮件是关于很多事情的（太笼统）

### 【关键信息】
提取标准：
1. 包含具体数据/时间/金额的句子
2. 表达决策或要求的句子
3. 涉及多方协作的说明
4. 重要变更或更新

格式：
- 使用 bullet points
- 每条不超过 80 字
- 最多 5 条

### 【时间信息】
必须包含：
- 发送时间（精确到小时）

可选包含：
- 会议时间
- 截止时间
- 有效期
- 提醒时间

### 【待办事项】
格式规范：
```
□ [具体行动] - [优先级] - [截止时间]
```

优先级标记：
- 高：红色/ urgent / 今天截止
- 中：黄色 / 本周截止
- 低：灰色 / 无明确时间

### 【优先级判断】
判断依据（按权重排序）：
1. 发件人身份（30%）
2. 关键词（25%）
3. 时间紧迫性（25%）
4. 任务数量（20%）

理由撰写：
- 简明扼要
- 列出主要因素
- 不超过 30 字

## 示例对比

### 原始邮件
```
From: 王经理 <wang@company.com>
Subject: 关于下周产品发布会的安排

大家好，

下周三（3月15日）下午2点我们将举行产品发布会。
请各部门做好准备：
- 技术部：确保演示环境稳定
- 市场部：准备宣传材料
- 销售部：邀请重要客户

请务必在本周五前确认参会人员。
如有问题请及时联系我。

谢谢！
王经理
```

### 标准摘要
```
📧 邮件摘要
━━━━━━━━━━━━━━━━━━━━━

【发件人】
王经理

【核心主题】
3月15日产品发布会安排及部门分工

【关键信息】
• 发布会时间：3月15日下午2点
• 技术部负责演示环境
• 市场部准备宣传材料
• 销售部邀请客户

【时间信息】
• 发送时间：2026-02-26 09:30
• 截止时间：本周五前确认参会人员

【待办事项】
□ 技术部准备演示环境 - 高 - 3月15日前
□ 市场部准备宣传材料 - 高 - 3月15日前
□ 销售部邀请客户 - 高 - 3月15日前
□ 确认参会人员名单 - 高 - 本周五前

【优先级判断】
高 - 有时间要求、涉及多部门协作

【建议行动】
• 立即确认本部门参会人员
• 着手准备相关工作
• 遇到问题及时反馈
```

## 常见错误

### ❌ 过于简略
```
【核心主题】
关于一个会议

【关键信息】
• 要开会
• 请准备
```

### ❌ 过于详细
```
【关键信息】
• 第一段的全部内容复制
• 第二段的全部内容复制
• ...（超过10条）
```

### ❌ 主观臆断
```
【优先级判断】
高 - 王经理总是发紧急邮件
```

### ✅ 正确示例
```
【优先级判断】
高 - 涉及跨部门协作且有明确截止时间
```
