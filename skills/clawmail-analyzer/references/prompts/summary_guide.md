# 摘要生成规则

## 输出字段

LLM 需要输出三个字段：

| 字段 | 限制 | 说明 |
|------|------|------|
| keywords | 3-8个 | 最具代表性的关键词，覆盖主题、人物、时间、金额等核心实体 |
| one_line | 30字以内 | 一句话核心概括，包含具体数字/日期/金额（如有） |
| brief | 2-3行，硬限100字 | 纯内容摘要，只写邮件说了什么，不评论邮件类型 |

## 基本原则

1. **信息密度高**：每句话都携带有效信息，省略礼貌用语和客套话
2. **客观准确**：不添加主观判断，不夸大或缩小事实
3. **保留关键数据**：数字、日期、金额、百分比等具体数据必须保留
4. **不得推断**：只概括原文明确表达的内容，不补充原文未提及的信息

## one_line 规则

- 概括邮件的**核心意图**，而非罗列内容
- 如有具体数字/日期/金额，必须包含
- 对于线程回复邮件，体现"回复了什么"而非重复主题行

好的例子：
- "张总要求周五前提交Q4财务报告"
- "产品上线延迟至3月底，需调整市场计划"
- "李四确认了500万预算并同意分两期拨付"

差的例子：
- "关于一个会议"（太笼统，无具体信息）
- "发了一封邮件"（无实质内容）
- "回复：Q4项目进度汇报"（只是重复主题行）

## brief 规则

- 2-3行，**硬限100字**（超过将被截断）
- 只写邮件**说了什么**，不要评论邮件类型（如"这是一封论坛转帖"）或是否需要回复
- 重要信息前置：核心请求/通知 → 关键细节（时间/数据/条件）
- 如果邮件内容简单，1-2行即可

好的例子：
```
张总要求各部门在周五（3月1日）前提交Q4季度财务报告，需包含收入、支出、利润三大板块数据，完成后抄送财务部审核。
```

差的例子：
```
这是一封来自张总的邮件。（←元评论，不是内容摘要）
张总在邮件中提到了一些关于报告的事情。（太模糊）
属于工作类邮件，需要及时回复。（←元评论，浪费字数）
```

## keywords 规则

- 3-8个，按重要性排序
- 覆盖：主题领域、关键人物/组织、时间节点、金额数据
- 不要提取过于泛化的词（如"邮件"、"通知"、"工作"）

好的例子：`["Q4报告", "财务数据", "周五截止", "张总"]`

差的例子：`["邮件", "工作", "通知", "重要"]`
