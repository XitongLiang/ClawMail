# 优先级判断标准

## 评分系统

总分为 10 分，根据以下维度评分：

### 1. 发件人身份（最高3分）

| 身份 | 分数 | 说明 |
|------|------|------|
| CEO/总经理/老板 | 3 | 最高级别 |
| 直接上级/部门经理 | 2 | 直属领导 |
| 重要客户/合作方 | 2 | 外部关键人物 |
| 其他部门经理 | 1 | 跨部门协作 |
| 普通同事 | 0 | 平级 |
| 系统通知 | 0 | 自动发送 |

### 2. 紧急关键词（最高3分）

#### 3分词（极度紧急）
- 紧急、urgent、ASAP、立即、马上、即刻、刻不容缓

#### 2分词（较紧急）
- 重要、important、尽快、今天、务必、必须、关键

#### 1分词（一般）
- 请、需要、麻烦、建议、希望

### 3. 时间紧迫性（最高2分）

| 时间要求 | 分数 |
|----------|------|
| 今天/立即/马上 | 2 |
| 明天/本周 | 1 |
| 下周及以后 | 0 |
| 无明确时间 | 0 |

### 4. 任务复杂度（最高2分）

| 情况 | 分数 |
|------|------|
| 3个及以上高优先级待办 | 2 |
| 1-2个高优先级待办 | 1 |
| 只有中低优先级待办 | 0 |

## 优先级划分

### 高优先级（7-10分）
**特征**：
- 来自高层或重要客户
- 包含紧急关键词
- 今天或立即需要处理
- 多个高优先级任务

**处理建议**：
- 立即查看并处理
- 暂停其他低优先级工作
- 如无法按时完成需立即反馈

### 中优先级（4-6分）
**特征**：
- 来自直接上级或常规客户
- 包含一般性行动要求
- 时间要求在一周内
- 少量待办事项

**处理建议**：
- 当天或次日处理
- 合理安排时间
- 提前准备避免延误

### 低优先级（0-3分）
**特征**：
- 来自普通同事或系统
- 无紧急关键词
- 无明确时间要求
- 资讯通知性质

**处理建议**：
- 空闲时处理
- 批量处理同类邮件
- 无需立即响应

## 判断示例

### 示例1：高优先级
```
发件人：CEO <ceo@company.com>
主题：紧急：明日董事会材料准备
内容：请务必在今天下班前准备好...

评分：
- 发件人：CEO = 3分
- 关键词：紧急、务必 = 3分
- 时间：今天 = 2分
- 任务：高优先级待办 = 2分
总分：10分 → 高优先级

理由：来自CEO、明确紧急要求、今日截止
```

### 示例2：中优先级
```
发件人：王经理 <wang@company.com>
主题：请确认下周会议时间
内容：需要您确认一下...

评分：
- 发件人：直接上级 = 2分
- 关键词：需要 = 1分
- 时间：下周 = 0分
- 任务：1个中优先级 = 0分
总分：3分 → 低优先级？再检查...

修正：
实际包含行动要求，调整至中优先级
理由：来自直接上级，需要确认行动
```

### 示例3：低优先级
```
发件人：系统通知 <noreply@company.com>
主题：月度报告已生成
内容：您的月度工作报告已生成...

评分：
- 发件人：系统通知 = 0分
- 关键词：无 = 0分
- 时间：无 = 0分
- 任务：无 = 0分
总分：0分 → 低优先级

理由：系统通知，无行动要求
```

## 特殊情况处理

### 误判修正
当出现以下情况，可人工调整优先级：

1. **已知背景**：虽然评分低，但已知此事很重要
2. **连锁影响**：此事延误会影响其他重要事项
3. **历史记录**：该发件人过去发送过紧急事项
4. **上下文**：结合其他邮件或消息判断

### 动态调整
优先级可能随时间变化：
- 今日的高优先级 → 明日变为中优先级
- 中优先级临近截止 → 升级为高优先级
- 事项已完成部分 → 降低优先级
