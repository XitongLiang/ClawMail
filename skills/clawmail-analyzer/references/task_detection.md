# 待办事项识别规则

## 触发词库

### 一级触发词（高置信度）
- 请 + 动词（请确认、请回复、请审核）
- 需要您 + 动词
- 务必 + 动词
- 麻烦 + 动词
- 期待 + 名词（期待回复、期待确认）

### 二级触发词（中置信度）
- 建议 + 动词
- 希望 + 动词
- 最好 + 动词
- 可以 + 动词

### 三级触发词（需上下文判断）
- 需要 + 名词
- 要求 + 名词
- 准备 + 名词

## 识别模式

### 模式1：直接指令
正则：`(?:请|务必|麻烦|需要您)\s*([动作].*?)(?:[，。；]|$)`

示例：
- "请于今日下班前提交报告" → 提交报告
- "务必确认参会人员" → 确认参会人员
- "麻烦准备一下PPT" → 准备PPT

### 模式2：期待回复
正则：`期待.*?([\u4e00-\u9fa5]+).*?(?:回复|反馈|意见)`

示例：
- "期待您的回复" → 回复邮件
- "期待您的宝贵意见" → 提供反馈意见

### 模式3：时间限制
正则：`(?:在|于|截止|截至).*?(\d{1,2}[月/]\d{1,2}[日号]?).*?前?`

示例：
- "请在3月15日前完成" → 完成...（截止时间3月15日）
- "截止日期为本周五" → ...（截止时间本周五）

### 模式4：条件触发
正则：`如果.*?，.*?请(.*?)(?:[，。；]|$)`

示例：
- "如果有问题，请及时联系我" → 联系发件人（如有问题）

## 优先级判定

### 高优先级信号
- 时间词：今天、立即、马上、尽快、urgent、ASAP
- 强调词：务必、必须、紧急、重要、关键
- 发件人：上级、客户、老板

### 中优先级信号
- 时间词：本周、下周、几天内
- 动词：确认、回复、审核、检查
- 场景：会议邀请、进度汇报

### 低优先级信号
- 时间词：有时间、有空、稍后
- 动词：了解、参考、看看
- 场景：资讯分享、通知公告

## 待办格式标准化

### 内容精简
- 保留核心动词和对象
- 删除礼貌用语
- 限制在 20 字以内

转换示例：
```
原文："请麻烦您看一下附件中的方案，然后给我一些反馈意见"
精简："审核附件方案并提供反馈"

原文："如果可以的话，希望您能参加明天的会议"
精简："确认是否参加会议"
```

### 截止时间提取
常见格式：
- "X月X日" → 2026-X-X
- "本周X" → 本周对应日期
- "明天" → 明日日期
- "下班前" → 今日17:00
- "周末前" → 本周五

### 完整示例

输入邮件片段：
```
请各部门注意：
1. 请于本周五前提交Q1季度报告
2. 需要您确认下个月的会议时间
3. 麻烦准备一下项目演示PPT
4. 如果有疑问请及时联系我
5. 建议提前测试演示环境
```

输出待办：
```
□ 提交Q1季度报告 - 高 - 本周五前
□ 确认会议时间 - 中 - 待定
□ 准备项目演示PPT - 中 - 会议前
□ 联系发件人（如有疑问）- 低 - 按需
□ 测试演示环境 - 低 - 会议前
```

## 过滤规则

### 排除项
- 纯信息通知（"本周五放假"）
- 已完成事项（"上周已完成"）
- 对第三方的要求（"请小明做某事"，非收件人）
- 过于模糊（"请处理一下"，无具体对象）

### 合并规则
- 相似内容合并（"请确认时间"+"请回复" → 确认并回复）
- 同一件事分步骤（"准备材料"+"提交材料" → 准备并提交材料）
