# 任务类型定义与处理策略

## 任务类型识别规则

### 1. 会议时间确认 (meeting_confirmation)

**触发关键词（标题/主题）：**
- 会议、会、开会
- 时间、时段
- 确认、协调、约
- 下午、上午、明天、下周

**触发模式：**
- `确认.*会议.*时间`
- `会议.*确认`
- `约.*时间`
- `明天.*下午.*会`

**处理策略：**
1. 提取邮件中的时间线索
2. 生成3个标准时间段选项（14-15点/15-16点/16-17点）
3. 提供"询问对方"和"跳过"选项
4. 用户选择后发送确认邮件

**回复模板变量：**
- `{first_name}` - 收件人名
- `{time_slot}` - 时间段
- `{date}` - 日期（明天/下周等）

---

### 2. 邮件回复 (email_reply)

**触发关键词：**
- 回复、答复
- 跟进、跟进
- RE:、Re:
- 回邮件

**处理策略：**
1. 分析邮件内容生成回复草稿
2. 提供"发送/编辑/跳过"三选项
3. "发送"直接发送AI生成的回复
4. "编辑"打开撰写窗口让用户修改

---

### 3. 跟进提醒 (follow_up)

**触发关键词：**
- 跟进、跟催
- 检查、查看
- 提醒、记得
- 待确认

**处理策略：**
1. 查询相关邮件/任务状态
2. 生成状态摘要
3. 提供"标记完成/推迟1天/跳过"选项

---

### 4. 通用任务 (generic)

**默认处理：**
- 标记为完成
- 推迟1天
- 跳过

---

## 优先级处理规则

| 优先级 | 弹窗超时时间 | 默认选项 |
|--------|-------------|---------|
| high | 60秒 | 第一个选项 |
| medium | 45秒 | 第一个选项 |
| low | 30秒 | 跳过 |

## 邮件回复模板库

### 会议确认

**简短版：**
```
{name}，

好的，{date}{time_slot}可以。

到时候见！

{name}
```

**详细版：**
```
{name}，

收到，{date}{time_slot}我有空，可以参会。

会议相关准备我会提前做好，届时准时参加。

如有变动请提前告知。

谢谢，
{name}
```

### 询问时间

```
{name}，

收到，{date}下午我有空。

请问您建议哪个时间段比较方便？
- 14:00-15:00
- 15:00-16:00
- 16:00-17:00

请告知您的偏好，谢谢！

{name}
```

### 通用回复

```
{name}，

收到，我会尽快处理。

如有疑问会及时联系您。

谢谢，
{name}
```

## 错误处理策略

### 无关联邮件
- 显示警告信息
- 提供"手动输入邮件内容"选项
- 或提供"跳过此任务"选项

### API 超时/失败
- 重试3次（间隔2秒）
- 仍失败则标记任务为 `in_progress`（不是完成）
- 记录错误日志供人工处理

### 弹窗超时
- 使用默认选项（通常是第一个）
- 记录日志："用户未响应，使用默认选项"
- 继续执行

### 邮件发送失败
- 不标记任务完成
- 更新任务描述添加错误信息
- 保留任务供人工处理
