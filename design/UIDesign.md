行为规则：
- 普通文件夹"删除邮件" → folder 改为"已删除"（软删除，仅本地操作）
- 已删除"彻底删除" → 本地 DB 删除 + 异步 IMAP 服务端删除（按 Message-ID 跨文件夹搜索）
- 草稿箱"彻底删除" → 仅本地 DB 删除，不同步服务端
- "移回收件箱" → folder 改回"INBOX"
- "置顶"后邮件始终排在列表最前（ORDER BY pinned DESC, received_at DESC）

草稿箱（📝 草稿箱）交互
────────────────────────────────────────

列表展示：
- 显示"致: <收件人名/邮箱>"，不显示发件人（发件人=自己，无意义）
- 样式：灰色（#888888）+ 斜体字体
- 双击草稿 → 打开 ComposeDialog，预填收件人/抄送/主题/正文
- 关联 draft_id，保存时执行 update_draft() 而非新建

删除行为：
- 彻底删除 → 直接从本地 DB 删除（不经过"已删除"文件夹，不同步服务端）

ComposeDialog 草稿功能：
┌──────────────────────────────────────────────┐
│  收件人：[                                 ]  │
│  抄送：  [                                 ]  │
│  主题：  [                                 ]  │
│  ┌──────────────────────────────────────────┐ │
│  │  正文区域（QTextEdit 或 QWebEngineView） │ │
│  └──────────────────────────────────────────┘ │
│  状态提示: "草稿已保存"                        │
│            [取消]  [保存草稿]  [发送]          │
└──────────────────────────────────────────────┘

保存行为：
1. 点击"保存草稿" → 立即保存，状态栏显示"草稿已保存"
2. 编写中每 60 秒自动静默保存（无弹框）
3. 关闭/取消/ESC 时，若有内容且从未保存过草稿，弹框询问：
   - [是] → 保存草稿后关闭
   - [否] → 直接关闭，不保存
   - [取消] → 继续编辑，恢复定时器
4. 已有 draft_id（已保存过）再关闭 → 无弹框，直接关闭
5. 发送成功 → 自动删除对应草稿

回复/转发模式保存：
- reply-area contenteditable innerHTML 作为 body_html 存入草稿
- 通过 runJavaScript 异步提取后落盘
